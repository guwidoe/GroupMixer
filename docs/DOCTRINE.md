# Doctrine

GroupMixer adopts `docs/reference/principles/AGENTIC_ENGINEERING_PRINCIPLES.md` as a **normative doctrine** for this repository.

## Meaning

- The principles in that file are not optional guidance.
- If legacy structure, convenience shortcuts, undocumented workflows, or local habits conflict with the doctrine, the doctrine wins.
- Architecture, operability, testing, interfaces, and refactoring decisions should be evaluated against that doctrine.

## Scope

This doctrine applies across the repo, including:

- `backend/core/`
- `backend/wasm/`
- `backend/api/`
- `webapp/`
- supporting docs, tests, workflows, and developer tooling
